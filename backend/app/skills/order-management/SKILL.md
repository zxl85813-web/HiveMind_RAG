# Order Management Skill (Intents 01, 03, 04)

## ⚡ 快速参考 (Quick Reference)

| 意图代码 | 业务场景 | 对应工具 |
|----------|----------|----------|
| **01** | 订单详情查询 | `crm_get_order(order_id, is_logged_in)` |
| **03** | 订单取消申请 | `oms_cancel_order(order_id)` |
| **04** | 地址变更请求 | `oms_update_address(order_id, new_address)` |

## 📋 完整规程 (Full Protocol)

### 第一步：意图识别与单号提取
1.  **识别意图**: 区分是单纯查询 (01)、取消 (03) 还是改地址 (04)。
2.  **提取单号**: 
    - 官网格式: `EFXX-123456` 或 `#12345`。
    - 亚马逊格式: `3-7-7` (如 112-9710404-9205034)。
3.  **单号校验**: 若单号格式明显不对，提示："您提供的单号格式似乎有误，官网单号通常以 EF 开头，请核对。"

### 第二步：登录与权限校验 (关键)
- **校验上下文**: 检查 `is_logged_in` 变量。
- **逻辑分支**:
    - **涉及敏感信息 (01/04)**: 若 `is_logged_in` 为 `False`，**必须**引导用户登录。
    - **话术引导**: "为了您的账户安全，查询详情/修改地址需要您先登录。请点击页面右上角的【登录】。"

### 第三步：执行动作
- 调用对应工具，并捕获 `ERROR_001` (未找到) 等异常。
- **多轮逻辑**: 若用户连续3次未提供有效单号，回复："抱歉，我还是没能识别到有效的单号。为您转接人工客服处理吧。"

## 🔒 强制准则 (Critical Rules)

| 规则 | 说明 |
|------|------|
| 登录优先 | 未登录状态下，禁止通过任何方式透露具体收货地址或商品 SKU。 |
| 亚马逊特殊处理 | 亚马逊订单通常无法在官网系统执行取消或改地址，若识别为亚马逊单号执行取消逻辑，应提示"请前往亚马逊后台操作"。 |
| 语气要求 | 商务、专业、安抚（特别是在订单取消或修改失败时）。 |
