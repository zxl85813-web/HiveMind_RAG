# OpenAPI Contracts (Frozen)

This directory holds the **frozen OpenAPI 3.x schemas** for each HiveMind
deployment SKU. They are the single source of truth for:

* Front-end & external SDK code generation (`openapi-typescript-codegen`,
  `openapi-generator`, etc.).
* Customer integration contracts — pin your client to a specific file.
* Schema diff reviews — `git diff` shows API surface changes between branches.

## Files

| File          | Mode    | What's exposed |
| ------------- | ------- | -------------- |
| `full.json`   | `full`  | RAG + Agent (all routes) — the default platform |
| `rag.json`    | `rag`   | Knowledge retrieval platform only (no Agent Swarm) |
| `agent.json`  | `agent` | Agent orchestration platform only (no RAG pipeline) |

Every schema includes a non-standard marker `info["x-hivemind-platform-mode"]`
identifying which `PLATFORM_MODE` produced it.

## Regenerate

The schemas are generated by [`backend/scripts/export_openapi.py`](../../backend/scripts/export_openapi.py).
Run from the `backend/` directory:

```powershell
# All three SKUs (one subprocess each, clean import graph)
python -m scripts.export_openapi --mode all

# Just one mode
python -m scripts.export_openapi --mode rag
python -m scripts.export_openapi --mode agent
python -m scripts.export_openapi --mode full
```

The script only imports the FastAPI `app` object and calls `app.openapi()` —
it does **not** start the application lifespan, so no database / Redis /
vector store dependencies are required.

## Conventions

* **Always commit regenerated files together with the route changes that
  produced them.** A PR that adds a route but forgets to refresh these JSONs
  will be rejected by review.
* **Do not hand-edit these files.** Any required tweaks belong in the FastAPI
  route definitions or in the script's post-processing.
* **Versioning** follows `APP_VERSION` in `backend/app/core/config.py`.
  Breaking schema changes require bumping `APP_VERSION` and recording the
  change in `docs/changelog/`.
