# 🧠 Context Migration Skill（长程上下文迁移）

## 核心理念

> 目标不是「总结历史」，而是「重建一个已经运行了很久的认知系统，在新窗口中恢复其稳定坐标」。

新窗口有效接管度 = **核心主轴重建** × **目标恢复度** × **偏好继承度** × **风格连续性** × **当前任务定位准确度**

所有步骤都围绕这个公式展开。

---

## 📚 Skill Assets（技能资产）

### 1. 信息源优先级指南（`library/source-priority.md`）
- 定义了从哪些信息源按什么顺序读取历史认知

### 2. 迁移输出模板（`library/migration-report-template.md`）
- 标准化的五段式输出结构，确保每次迁移的报告格式一致、可比较

---

## 🔄 执行步骤（五阶段迁移法）

### 阶段 0：信息采集（Information Gathering）

**按优先级顺序读取以下信息源：**

1. **KI 系统（最高优先级）**
   - 通过对话开始时提供的 `<conversation_summaries>` 确认相关 KI
   - 使用 `view_file` 读取 `/brain/<conversation-id>/artifacts/` 下的关键文件
   - 重点关注：架构决策、偏好记录、方法论文档

2. **TODO.md（任务状态）**
   - 路径：`c:\Users\linkage\Desktop\aiproject\TODO.md`
   - 提取：未完成任务（`[ ]`）、最近完成任务（`[x]`）、任务优先级

3. **情节记忆 (Episodic Memory & Cross-session Summaries)**
   - 调用 `recall_episodes` 接口召回跨会话的情节摘要（基于语义和主题）
   - 比原始对话记录优先级更高，因为这是经过蒸馏的稳定认知
   - 重点：提取用户的决策轨迹、被反复强调的偏好、已蒸馏的「认知主轴」

4. **项目文档（按需）**
   - `docs/requirements/` 中的活跃需求文档（REQ-NNN）
   - `docs/design/` 中最近被修改的设计文档

**读取时执行「价值密度分层」（对应 `library/source-priority.md`）：**
- **核心长期信息**：决策、架构规则、用户偏好、方法论禁区
- **中层任务信息**：当前 Sprint 任务、进行中的 OpenSpec Change
- **临时过程信息**：调试日志、临时测试结果
- **可忽略噪声**：闲聊、重复内容、已撤销的决策

---

### 阶段 1：主轴提炼（Axis Extraction）

从采集的信息中识别：

| 维度 | 提炼目标 | 权重规则 |
|------|----------|----------|
| **核心主题** | 项目正在解决的最核心问题 | 高频出现 = 高权重 |
| **长期目标** | 用户最终想达到的状态 | 反复强调 = 稳定偏好 |
| **关键概念** | 项目特有的术语和框架 | 有定义 = 保留原文 |
| **偏好模式** | 用户对工作方式的稳定倾向 | 多次被强调 = 不可违背 |
| **风格特征** | 沟通深度、技术细节粒度、语言偏好 | 被修正过的版本优先 |
| **方法论** | 已形成的工作流、公式、框架 | 视为核心资产，不可随意替换 |
| **禁区与反感点** | 用户明确拒绝的做法 | 一旦发现，永久记录 |

---

### 阶段 2：连续性重建（Continuity Reconstruction）

明确回答以下五个连续性问题：

1. **之前已经讨论过什么？**（事实层）
2. **为什么会讨论这些？**（动机层——这是最容易被遗漏的）
3. **用户真正重视什么？**（价值层）
4. **现在最该继续哪一段工作？**（任务定位层）
5. **如何避免质量下降、风格变浅、理解跑偏？**（防衰退层）

---

### 阶段 3：输出迁移报告（Migration Report Output）

按 `library/migration-report-template.md` 中的标准模板输出：

```
## 🧠 上下文迁移报告

### 1. 核心主轴
[项目和对话系统正在干的最核心的事，1-3句话]

### 2. 高权重概念与稳定偏好
[最关键的概念、已形成的方法论、禁区]

### 3. 当前进度与位置
[最近完成了什么 → 现在卡在哪里 → 下一步最清晰的推进方向]

### 4. 后续接续策略
[新窗口应如何继续，包括：应优先完成的任务、需要重点注意的风格和偏好]

### 5. 迁移质量自评
[对照接管公式，逐项自评: 主轴重建/目标恢复度/偏好继承/风格连续性/任务定位]
```

---

### 阶段 4：直接接续（Seamless Continuation）

**迁移完成后，不停留在报告层！**

- 报告输出后，立即提出「下一步最高价值的具体行动」
- 如果有明确未完成的任务，直接进入执行状态
- 保持已继承的语言气质、深度和推进方式

---

## ⚠️ 执行禁忌（Red Flags）

- **禁止流水账摘要**：不是在复述历史，是在重建认知坐标
- **禁止只看表面主题**：必须穿透到「用户为什么在意这个」的动机层
- **禁止跳过偏好层**：即使时间紧，也必须识别已稳定的工作模式偏好
- **禁止停留在报告层**：迁移是手段，接续工作才是目的
- **禁止新窗口降质**：如果旧窗口的讨论有深度，新窗口必须维持同等深度

---

## 🔗 协同关系

- **输入来源**：对话摘要 + KI 系统 + TODO.md + docs/
- **触发时机**：每次新会话开始时（可选手动触发）
- **输出终点**：迁移报告 → 直接接续执行（可选对接 `openspec-explore` 或 `develop-feature`）
