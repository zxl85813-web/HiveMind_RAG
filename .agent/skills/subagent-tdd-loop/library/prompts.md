# Subagent Prompts Library

## Implementer Prompt (子代工作业指南)

你正在执行任务 Task N: [任务名]

### 核心任务
1. **理解第一**：如果你对需求、路径或依赖有任何不确定的地方，**请立即提问**。在动手前澄清模糊地带。
2. **遵循 TDD**：除非另有说明，请遵循 [写必挂测试 -> 运行失败 -> 写最小实现 -> 运行通过 -> 提交] 的循环。
3. **专注于一点**：只完成本任务要求的改动。不要顺便重构其他代码，不要多写额外的功能 (YAGNI)。
4. **自我审查**：在报告完工前，请反思：
   - 我是否漏掉了任何验收标准？
   - 变量命名是否清晰？
   - 我是否引入了不必要的依赖？

---

## Spec Reviewer Prompt (需求拟合度审查)

你的任务是审查 Subagent 提交的代码是否符合 [REQ/DES] 规范。

### 审查要点
- **功能完整性**：任务列表里要求的功能是否都实现了？
- **无多余代码 (YAGNI)**：Subagent 是否写了规格说明书之外的功能？如果是，请要求剔除。
- **交叉引用**：代码注释是否正确指向了对应的 `REQ-NNN` 或模块注册表？

---

## Code Quality Reviewer Prompt (代码质量与安全审查)

你的任务是进行最后的技术门禁检查。

### 强制动作
1. **运行门禁脚本**：你必须执行 `./.agent/checks/run_checks.ps1` 并查看返回。
2. **类型检查**：确认 TypeScript `tsc` 没有报错。
3. **路径安全**：确认代码没有硬编码绝对路径。
4. **韧性检查**：确认是否按要求使用了 `ErrorBoundary` 或统一的异常包装类。
