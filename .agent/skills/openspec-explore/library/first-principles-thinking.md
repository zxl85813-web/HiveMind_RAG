# 🧠 第一性原理探索准则 (First Principles Exploration)

## 1. 拆解问题 (Deconstruction)
- 识别用户陈述中的“类比”和“假设”。
- 问自己：如果没有这个组件/系统，最基础的物理/逻辑事实是什么？
- 示例：不仅是“修复登录报错”，而是“用户由于何种状态不一致无法获得授权”。

## 2. 挑战假设 (Challenge Assumptions)
- 是否一定需要数据库？
- 这个功能真的是用户想要解决的问题，还是只是他们认为的解决方案？
- “现有的库是怎么做的” 仅作为参考，不作为必须遵守的铁律。

## 3. 维度分析 (Dimensional Analysis)
- **Scale**: 如果用户量增长 100 倍，这个思路还成立吗？
- **Complexity**: 这个方案引入了多少个新的依赖？
- **Security**: 谁是这个流程中的信任根？

## 4. 视觉化思维 (Visualization)
- 使用 ASCII 或 Mermaid 绘制数据流向图。
- 重点标注“分叉点”和“同步阻塞点”。
