# 更新 TODO 清单 — 强制流程

> **触发时机**: 每次开发对话即将结束时（用户不再提出新需求时），必须执行此流程。
> **责任人**: AI 助手 (自动执行)

## 📋 步骤

### 1. 回顾本次对话中完成的工作
- 列出所有已完成的功能、修复的 Bug、新增的文件
- 检查是否有代码层面的遗留问题（如 TODO 注释、临时写法）

### 2. 收集未完成 / 推迟的事项
重点关注以下类型的内容：
- 💬 对话中提到"以后再做"、"先跳过"、"暂时不管"的事项
- 🐛 发现但未修复的 Bug
- ⚠️ 用了临时方案的地方（如 Mock 数据、硬编码值、简单实现）
- 📝 讨论过但未落地的架构决策
- 📦 需要安装但未安装的依赖

### 3. 更新 TODO.md
打开 `TODO.md`，按以下规则更新：

#### 3a. 更新"最后更新时间"
将文件顶部的日期更新为当前日期。

#### 3b. 已完成的事项
将对应条目的标记改为 ✅，并在"变更日志"章节追加记录。

#### 3c. 新发现的 Bug
追加到"已知问题 (Known Issues)"表格中，格式：
```
| BUG-XXX | 问题描述 | 相关文件 | 发现时间 | 🐛 待修 |
```

#### 3d. 推迟 / 搁置的事项
- 如果是新功能，追加到对应的功能章节
- 如果是暂时跳过的问题，标记为 ⏸️ 并写明原因
- 如果是讨论过的决策，追加到"架构与规划"章节

#### 3e. 新增依赖
追加到"依赖安装备忘"章节

### 4. 输出摘要
在对话最后，用以下格式输出本次的变更摘要：

```
📋 TODO 更新摘要:
- ✅ 已完成: [X 项]
- 🐛 新发现 Bug: [X 项]
- ⏸️ 暂时搁置: [X 项]
- ⬜ 新增待做: [X 项]
```

## ⚠️ 注意事项

1. **不要遗漏**: 即使是很小的问题（如一个没翻译的字符串），也要记录
2. **写清上下文**: 每个条目必须包含：问题描述 + 相关文件 + 为什么推迟
3. **不要过度乐观**: 如果只完成了 80%，标记为 🟡 而不是 ✅
4. **保持编号连续**: Bug 编号、决策编号不要跳号
