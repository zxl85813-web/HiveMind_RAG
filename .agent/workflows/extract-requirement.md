# 📋 需求提取流程

当用户在对话中提出新的功能需求或变更时，执行此流程。

## 步骤

### 1. 识别需求
在对话中识别出明确的功能需求或变更要求。

### 2. 检查已有需求
// turbo
```bash
dir docs\requirements\
```
确认没有重复的需求文档。

### 3. 分配编号
查看现有 REQ-NNN 文件，分配下一个编号。

### 4. 创建需求文档
按以下模板创建 `docs/requirements/REQ-NNN-slug.md`:

```markdown
# REQ-NNN: {标题}

| 字段 | 值 |
|------|------|
| **编号** | REQ-NNN |
| **标题** | {标题} |
| **来源** | 用户对话 ({日期}) |
| **优先级** | 高/中/低 |
| **状态** | ⬜ 待讨论 / 🟡 设计中 / 🟢 方案已定 / ✅ 已实现 |
| **关联设计** | DES-NNN (如有) |
| **关联代码** | {文件路径} |

## 需求描述
{从对话中提取的需求}

## 详细要求
1. ...
2. ...

## 验收标准
- [ ] ...

## 变更记录
| 日期 | 变更 | 人员 |
|------|------|------|
| {日期} | 初始创建 | AI |
```

### 5. 更新 REGISTRY.md
在注册表的 traceability 部分登记新需求。

### 6. 通知用户
告知用户需求已归档，确认需求理解是否正确。
