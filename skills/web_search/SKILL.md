# Web Search Skill

## Overview

从互联网获取实时信息来增强 Agent 回答。核心原则：**搜索是补充手段，不是默认行为** — 优先使用知识库，仅在知识库无法满足时启用网络搜索。

## Quick Reference

| 任务 | 工具 | 说明 |
|------|------|------|
| 网络搜索 | `web_search` | 调用搜索 API 获取结果列表 |
| 网页抓取 | `fetch_page` | 提取特定 URL 的正文内容 |
| 内容摘要 | `summarize_page` | 对抓取内容生成结构化摘要 |
| 多源验证 | `cross_verify` | 对同一事实查找多个独立来源 |
| 学术搜索 | `search_academic` | 搜索学术论文和技术文档 |

## 搜索策略

### 1. 何时启用网络搜索

**决策树：**

```
用户提问 → 知识库有相关内容？
    ├─ 是 → 使用 RAG 检索（rag_search skill）
    │        └─ RAG 结果置信度足够？
    │            ├─ 是 → 直接回答，不搜索
    │            └─ 否 → 补充网络搜索
    └─ 否 → 问题需要实时信息？
         ├─ 是 → 使用网络搜索
         │        └─ 如：新闻、股价、天气、最新版本
         └─ 否 → 使用模型内置知识
                  └─ 如：通用概念、历史事实
```

**必须搜索的场景：**
- 涉及日期、时间的实时信息（"今天的..."、"目前..."、"最新..."）
- 具体产品/库的版本号和更新
- 新闻事件和行业动态
- 竞争对手分析等商业情报
- 用户明确要求搜索

**不应搜索的场景：**
- 知识库已有全面覆盖的内容
- 通用编程概念和基础知识
- 用户未授权外部网络访问时

### 2. 查询构造

**搜索查询 ≠ 用户提问。** 需要从对话意图中提取最佳搜索关键词。

```python
# ❌ 错误 — 直接用自然语言问句搜索
results = await web_search("HiveMind 怎么配置多个 LLM 模型？")

# ✅ 正确 — 提取关键词 + 限定条件
results = await web_search(
    query="LLM multi-model routing configuration",
    time_range="past_year",      # 限制时间范围
    site_filter="docs.langchain.com OR github.com",  # 限定来源
    num_results=10,
)
```

**查询优化技巧：**
- **使用英文搜索技术内容** — 英文技术文档质量通常更高
- **添加时间限定** — 技术领域变化快，优先搜最近 1 年内容
- **限定高质量来源** — 官方文档 > 技术博客 > 问答社区 > 通用网站
- **使用引号精确匹配** — 搜索特定错误信息或术语时

### 3. 多轮搜索策略

复杂问题通常需要多轮搜索：

```python
# 第 1 轮：广泛搜索，了解主题轮廓
broad_results = await web_search("vector database comparison 2025")

# 分析结果，识别关键维度
dimensions = extract_comparison_dimensions(broad_results)

# 第 2 轮：针对每个维度深入搜索
for dim in dimensions:  # 如：性能、成本、生态
    detail = await web_search(f"vector database {dim} benchmark 2025")
    
# 第 3 轮：交叉验证关键事实
key_claims = extract_key_claims(all_results)
for claim in key_claims:
    verification = await cross_verify(claim)
```

## 网页内容提取

### 抓取策略

```python
# 直接抓取正文内容
content = await fetch_page(
    url=result.url,
    extract_mode="article",    # 只提取正文，去广告/导航
    max_length=5000,           # 限制内容长度
    timeout=10,                # 超时秒数
)
```

**抓取优先级：**
1. **官方文档** — 最权威，优先抓取
2. **GitHub README/Wiki** — 技术细节丰富
3. **技术博客** — 有实战经验
4. **Stack Overflow 回答** — 注意区分高赞/低赞
5. **通用网页** — 仅作为补充

### 内容质量评估

对每个抓取的内容评估可信度：

| 来源类型 | 可信度 | 处理方式 |
|----------|--------|----------|
| 官方文档 | 🟢 高 | 可直接引用 |
| 知名技术博客 | 🟢 中高 | 可引用，注明来源 |
| 社区回答 (高赞) | 🟡 中 | 需交叉验证 |
| 个人博客 | 🟡 中低 | 参考但需验证 |
| 未知来源 | 🔴 低 | 仅作背景信息 |

## 结果整合与呈现

### 回答格式

```markdown
## 回答

[基于多个来源的综合回答]

### 来源
1. [来源标题](URL) — 官方文档 🟢
2. [来源标题](URL) — 技术博客 🟡
3. [来源标题](URL) — GitHub Issue 🟡

> ⚠️ 最后搜索时间：2026-02-16。信息可能已过时，建议验证关键数据。
```

### 信息冲突处理

当多个来源信息矛盾时：

```markdown
⚠️ **注意：** 关于 [主题]，不同来源存在分歧：
- **来源 A (官方文档)** 称：[观点 A]
- **来源 B (技术博客)** 称：[观点 B]

**建议以官方文档为准。** 来源 B 可能基于旧版本。
```

## 安全与隐私

### 搜索安全规则

- **不搜索用户的敏感信息** — 如密码、API Key、个人隐私
- **不访问需要认证的页面** — 只抓取公开可访问的内容
- **记录搜索日志** — 所有搜索查询和结果需可追溯
- **尊重 robots.txt** — 不抓取明确禁止的内容
- **限制请求频率** — 避免对目标网站造成压力

### 内容过滤

```python
# 过滤不可靠或有害内容
filtered = filter_results(
    results,
    block_domains=["spam-site.com"],        # 黑名单域名
    require_https=True,                      # 只接受 HTTPS
    min_content_length=200,                  # 过滤低质内容
    max_age_days=365,                        # 过滤过旧内容
)
```

## 常见陷阱

- **不要默认搜索** — 先检查知识库，知识库无法满足才搜索
- **不要信任单一来源** — 重要事实至少 2 个独立来源确认
- **不要忽略时效性** — 技术内容过期很快，标注搜索时间
- **不要搜索过于宽泛** — 精确的搜索词比广泛的搜索词效果好
- **不要暴露搜索过程** — 用户关心的是答案，不是搜索过程细节
- **不要忽略搜索失败** — 搜不到也是有用信息，告知用户并建议替代方案
