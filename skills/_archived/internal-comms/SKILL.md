## When to use this skill

撰写内部沟通文档时使用，支持以下类型：
- 项目状态更新（进展 / 计划 / 阻塞）
- 团队周报和月报
- 技术决策记录 (ADR)
- FAQ 和常见问题解答
- 事故报告和复盘
- 项目汇报（面向管理层）

## How to use this skill

撰写任何内部沟通文档时：

1. **识别沟通类型** — 从用户请求中判断
2. **加载对应的指南文件** — 从 `examples/` 目录获取：
    - `examples/3p-updates.md` - 进展/计划/问题 (Progress/Plans/Problems) 团队更新
    - `examples/company-newsletter.md` - 团队/公司通讯
    - `examples/faq-answers.md` - FAQ 和常见问题解答
    - `examples/general-comms.md` - 其他不属于以上类型的通用沟通
3. **遵循指南中的具体指示** — 关于格式、语气和内容组织

如果沟通类型不匹配任何现有指南，请向用户请求更多上下文或格式偏好。

## HiveMind 沟通风格

### 语气
- **直接简洁** — 避免冗余废话，直奔主题
- **数据驱动** — 用数字说话，避免模糊描述
- **行动导向** — 每个更新都包含下一步行动

### 格式规范
- 使用 Markdown 格式
- 标题层级不超过 3 级
- 列表优先于长段落
- 关键数据用 **粗体** 标注
- 时间线和里程碑使用表格
