## 需求描述

系统应有订阅机制，能根据当前技术架构和需求，在互联网实时获取最新技术动态（包括高星开源项目和可用的 Skill），并反馈给用户。

## 详细要求

1. **技术栈感知** — 自动根据当前技术架构生成监控关键词
2. **多源获取** — GitHub Trending、HN、ArXiv、PyPI、npm 等
3. **相关性评估** — LLM 评估与当前项目的相关度
4. **用户订阅** — 支持用户自定义订阅主题
5. **主动推送** — 高价值发现通过 WebSocket 推送
6. **定期报告** — 生成技术动态周报

## 验收标准

- [ ] 能自动获取 GitHub Trending 项目
- [ ] 相关性评分准确率 > 70%
- [ ] 推送延迟 < 1 分钟 (从发现到通知)
- [ ] 用户可管理订阅列表
