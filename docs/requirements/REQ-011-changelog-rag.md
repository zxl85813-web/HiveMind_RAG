## 需求背景
在处理需求说明书、设计文档（Word/Excel）等文档时，文档开头通常包含“变更履历表”。
传统 RAG 的分块策略会将履历表和文档正文割裂，导致无法准确回答“某段时间内的变更汇总”或“某个具体需求的变更历史”。
本需求旨在建立一套能够感知并提取变更履历、并将其与正文内容关联的 RAG 入库与检索体系。

## 详细要求
1. **变更履历解析 (Changelog Extraction)**:
    - 能够识别 Word/Excel 中的变更履历表格（日期、版本、作者、变更内容、关联章节）。
    - 将每条变更提取为独立的 `ChangeEvent` 原子单元。
2. **上下文缝合 (Context Stitching)**:
    - 建立变更项与文档正文章节的映射关系。
    - 在正文分块 (Chunk) 时，自动将关联的变更时间、版本号、作者等信息注入到 Metadata。
3. **结构化检索 (Structured Retrieval)**:
    - 支持按时间范围（date_from, date_to）过滤变更汇总。
    - 支持按版本号、作者等 Metadata 精确检索。
    - 检索结果应能同时展示变更摘要和对应的详细正文内容。

## 验收标准
- [ ] 能够准确解析 Excel 中的变更履历表并转为结构化数据。
- [ ] 内容 Chunk 的 Metadata 中包含对应的变更版本和日期。
- [ ] 支持查询“2025年1月10日到2025年2月10日之间的所有变更”。
- [ ] 检索“用户登录”需求时，能带出其最后一次变更的时间和作者。
