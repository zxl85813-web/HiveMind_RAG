## 需求描述

为了降低普通用户创建和管理高质量 Agent 的门槛，系统需要提供一个 **Agent 辅助构建工具**（建议以 Chat 模式深度集成至前端 UI 的 Agent Studio 或工作台中）。该助手将充当用户的 "Agent 架构师"，从需求分析、配置开发、效果评估、对比测试以及资源消耗（Tokens）等全生命周期提供智能化辅助，并与现有的图谱（GraphRAG / 思考链路执行图）进行深度结合。

## 详细要求

### Phase 0: 需求发现与智能访谈 (Requirements Discovery) — ⭐ 最高优先级

> 这是整个工具的核心差异化能力 — 不做"表单填写器"，而是做"主动思考的 Agent 架构师"。

1. **结构化访谈对话 (Structured Interview)**
   - 通过多轮交互式对话（Chat 模式），以五阶段协议引导用户逐步澄清目标与场景。
   - 五阶段: 角色澄清 → 上下文注入 → 外部调研 → 差距分析 → 确认生成。
   - 每个阶段有明确的"完成条件"，覆盖率不足时自动回退到更早阶段。

2. **多源上下文注入 (Multi-Source Context Injection)**
   - 后台自动并行查询: 对话记忆 (`SharedMemoryManager`)、知识库 (`RAGGateway`)、知识图谱 (Neo4j)、技能注册表 (`SkillRegistry`)、已有 Agent 列表 (`SwarmOrchestrator`)。
   - 将发现转化为自然语言建议呈现给用户（如："我发现系统中已有一个相关知识库…"）。

3. **外部调研与对标 (Web Research & Benchmarking)**
   - 使用 `web_search` 工具主动搜索行业最佳实践、竞品 Agent 设计模式。
   - 将搜索结果提炼为可操作建议呈现给用户。

4. **主动差距分析 (Proactive Gap Analysis)**
   - 内置 10+ 维度检查清单（安全护栏、异常回退、成功指标、测试场景、工具权限、多语言、扩展性等）。
   - 当覆盖率 < 70% 时，助手主动挑选最关键的未覆盖维度向用户追问。
   - 用 `think` 工具在内部做自省推理，确保追问的问题有针对性。

5. **需求摘要卡片 (Requirements Summary Card)**
   - 访谈完成后，生成一张结构化的需求摘要卡片，前端渲染为可编辑表单。
   - 用户可在卡片上直接修改，修改后触发配置重新生成。

### Phase 1: 配置生成与开发集成

1. **System Prompt 自动生成**
   - 自动生成高质量的 System Prompt、核心职责描述。
   - 自动推荐或挂载最相关的知识库 (Knowledge Base) 和工具 (MCP Skills/Tools)。

2. **开发与配置集成 (Development & Config)**
   - 提供"一键应用"功能，助手在对话中生成的配置可直接应用到系统的 Agent 创建界面中，实现 UI 联动。
   - 在图谱中自动规划并展示 Agent 的依赖关系（它会调用哪些工具，查阅哪些知识库）。

### Phase 2: 评估、测试与监控

1. **评估与预演 (Evaluation & Sandbox)**
   - 提供沙箱环境供用户进行预演测试。
   - 结合系统现有的 RAG/Agent 评估体系（如 M5 里程碑中的 RAGAS），对 Agent 处理典型 Case 的效果进行打分。

2. **测试对比 (A/B Testing)**
   - 提供双屏或多维度对比能力。允许用户输入同一条指令，对比"旧版提示词"与"助手优化的提示词"的执行效果差异。
   - 对比不同模型（如 Gemini 3.1 Pro vs 其他模型）在同一 Agent 设定下的表现。

3. **Tokens 消耗预估与监控 (Tokens & Cost)**
   - 在构建和沙箱测试期间，实时展示 Token 消耗量及预估成本。
   - 提供优化建议（如：提示词过长、上下文压缩建议等）以降低实际运行时的 Tokens 消耗。

4. **结合 Graph 与可视化**
   - 展现 Agent 的能力图谱（如：它与知识节点的连接图）。
   - 在测试时，通过执行 DAG（有向无环图）直观展示 Agent 每一步的决策树、工具调用链路和推理过程。

## 验收标准

### Phase 0 验收
- [ ] 助手能在对话中主动追问用户至少 3 个未提及的关键维度（如安全护栏、异常回退等）。
- [ ] 助手能自动发现系统中已有的相关 KB 和 Agent，并在对话中以自然语言形式建议挂载。
- [ ] 助手能使用 web_search 搜索行业最佳实践，并将发现整合到建议中。
- [ ] 访谈完成后，生成的需求摘要卡片覆盖率 ≥ 70%（10 个核心维度中至少 7 个已确认）。

### Phase 1 验收
- [ ] 助手能够通过 Chat 界面引导用户从零开始生成一个完整的 Agent 配置文件。
- [ ] 生成的配置能够直接联动覆盖到前端 Agent 创建表单。

### Phase 2 验收
- [ ] 支持在沙箱中对生成的 Agent 进行模拟对话，并输出简单的评估报告。
- [ ] UI 上提供明显的 Token 消耗统计与 A/B 对比面板。
- [ ] 支持在执行预演时渲染工具调用与思维链路图谱 (DAG)。
