## 需求描述

建立完整的开发治理体系，确保 AI 辅助开发的代码质量和一致性。

## 详细要求

### 文档体系
1. **需求文档** — 从对话中实时提取需求，形成 REQ-NNN 文档
2. **设计文档** — 架构与方案设计，形成 DES-NNN 文档
3. **API 文档** — 前后端 API 完整文档
4. **代码与文档绑定** — 通过交叉引用实现可追溯

### 代码规范
1. **结构化** — 固定目录结构，禁止随意添加
2. **组件共通化** — 功能注册表防止重复造轮子
3. **一致性** — 统一框架和库的使用方式
4. **样式规范** — Design Token + Ant Design 组件规范

### 代码质量
1. **代码分解** — 复杂功能必须分解为子任务
2. **过程日志** — 开发过程记录在 changelog/devlog/
3. **自省 (Review)** — 版本milestone 时进行代码审查
4. **自查 (Lint + Test)** — 代码规范审核 + 测试用例生成执行
5. **非实时** — 自省自查在达到版本里程碑时进行，非每次提交

### DevOps (后期)
1. **GitLab** — 公司内网 CI/CD pipeline
2. **GitHub** — 外网 CI/CD pipeline
3. **自动化** — lint → test → build → deploy
