# 🛡️ Governance Incident: [INC-20260412-003] 
# 前后端角色模型不一致导致菜单大规模缺失 (Role Schema Mismatch)

---

## 1. 事故描述 (Description)
- **触发点**: 用户（Role: `user`）登录后，侧边栏菜单缺失严重，仅剩余 Dashboard 等基础页面。
- **症状**: 
    1. 绝大部分功能页面（评测、批处理、设置等）消失。
    2. 实际上用户在后端拥有相应权限，但在前端无法渲染。

## 2. 根因分析 (Root Cause Analysis)

### 2.1 Schema 漂移 (Schema Drift)
- **后端模型**: `admin`, `user`, `readonly`。
- **前端模型**: `admin`, `operator`, `viewer`。
- **映射逻辑**: 前端在处理后端返回的 `user` 角色时，由于无法匹配 `operator`，默认降级为了 `viewer`。而在前端定义中，`viewer` 的权限集合为空 (`[]`)，导致所有受保护路由均被拦截并隐藏。

## 3. 治理动作 (Remediation)
- **对齐模型**: 修改了前端 `authStore.ts` 和 `access.ts`，将角色重构为 `admin`, `user`, `readonly`，与后端 SSoT (Single Source of Truth) 对齐。
- **权限重映射**: 为 `user` 角色补全了 `knowledge:manage`, `learning:manage`, `evaluation:run`, `batch:operate` 等合法的 UI 权限。
