# Self Reflective Learning System

## Goal

Build a daily self-learning loop for HiveMind that:

- studies system documents every day
- studies GitHub Project and issue signals every day
- outputs user-facing learning notes
- outputs actionable system improvement suggestions

## Agent + Skill Interpretation

The learning report interpretation is now generated by an `Agent + Skill` pipeline:

- Agent: `LearningAnalystAgent`
- Skills:
	- `TrendClusteringSkill` (learning track clustering)
	- `VendorLandscapeSkill` (multi-vendor AI landscape interpretation)
	- `ImprovementPrioritizationSkill` (priority recommendations)

Implementation:

- `backend/app/services/learning_agentic.py`
- integrated by `backend/app/services/learning_service.py`

Output section in daily report:

- `Agent+Skill 深度解读`
- richer recommendation items with priority labels (`P0/P1/P2`)
- `传统RAG优化抓手`（混合检索、重排、分块、查询改写、离线评测等）

Deep interpretation model:

- `deepseek-v3-2-251201` (configured via `ARK_MODEL`)
- Interpretation is not limited to latest trends; it explicitly includes traditional RAG optimization playbook items.

## Daily Loop

1. Read system core docs (`docs/SYSTEM_OVERVIEW.md`, `docs/architecture.md`, governance docs).
2. Pull GitHub Project V2 items (if configured).
3. Pull recent open issues from repository.
4. Pull diversified external signals:
	- X watchlist (e.g. OpenAI/Anthropic/DeepMind/Meta/xAI/Qwen)
	- AI company official feeds
	- major AI ecosystem GitHub release signals
5. Generate structured suggestions based on risk signals.
5. Write markdown report to `docs/learning/daily/YYYY-MM-DD.md`.

## API

### Trigger Daily Learning

- `POST /api/v1/learning/daily-cycle`

Returns:

- report path
- counts for local materials / project items / issues
- generated suggestions

### List Reports

- `GET /api/v1/learning/daily-reports?limit=7`

Returns recent report file paths.

### Read Report Content

- `GET /api/v1/learning/daily-report-content?report_path=docs/learning/daily/YYYY-MM-DD.md`

Returns markdown content for frontend preview.

## Daily Automation

Use script:

- `backend/scripts/run_daily_learning_cycle.py`
- `backend/scripts/run_daily_learning_cycle_with_retry.py`
- `backend/scripts/register_daily_learning_task.ps1`

Run manually:

```powershell
python backend/scripts/run_daily_learning_cycle.py
python backend/scripts/run_daily_learning_cycle_with_retry.py --retries 3 --delay-seconds 20
```

Windows Task Scheduler example:

- Trigger: daily at 09:00
- Program: `python`
- Args: `backend/scripts/run_daily_learning_cycle.py`
- Start in: repository root

One-command registration (PowerShell as Administrator):

```powershell
powershell -ExecutionPolicy Bypass -File backend/scripts/register_daily_learning_task.ps1
```

Runtime logs:

- `docs/learning/daily/logs/YYYY-MM-DD.log`

## Required Configuration

Set in backend `.env`:

```env
GITHUB_TOKEN=...
GITHUB_REPO_OWNER=zxl85813-web
GITHUB_REPO_NAME=HiveMind_RAG
GITHUB_PROJECT_OWNER=zxl85813-web
GITHUB_PROJECT_NUMBER=1
SELF_LEARNING_REPORT_DIR=docs/learning/daily
SELF_LEARNING_ISSUE_LIMIT=20
SELF_LEARNING_X_ACCOUNTS=OpenAI,AnthropicAI,GoogleDeepMind,MetaAI,xai,QwenLM
SELF_LEARNING_AI_FEEDS=https://openai.com/news/rss.xml,https://www.anthropic.com/news/rss.xml,https://deepmind.google/discover/blog/rss.xml
SELF_LEARNING_GITHUB_WATCH_REPOS=openai/openai-python,anthropics/anthropic-sdk-python,google-gemini/generative-ai-python,huggingface/transformers,langchain-ai/langchain,microsoft/autogen

# ARK 深度解读模型 (Volcengine Ark)
ARK_API_KEY=...
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_MODEL=doubao-1-5-pro-32k-250115
```

Notes:

- If `GITHUB_PROJECT_OWNER` or `GITHUB_PROJECT_NUMBER` is missing, Project ingestion is skipped.
- If `GITHUB_TOKEN` is missing, GitHub data is skipped, but local doc learning still works.
- If ARK config is missing or unavailable, the system falls back to deterministic interpretation output.

## For System Users

Users can read the generated daily report and follow:

- what changed in the system knowledge map
- what changed in GitHub project/tasks
- what improvements are recommended today

This makes learning and improvement transparent and continuous.
