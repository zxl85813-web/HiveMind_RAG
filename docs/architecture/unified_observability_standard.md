# HiveMind 全链路可观测性标准提案 (OBS-MANIFESTO-V1)

## 1. 核心哲学 (Core Philosophy)
在 HiveMind 架构中，日志不再是“文本碎片”，而是**“语义化资产”**。任何一段代码运行时的状态，必须能够通过一个 `trace_id` 拼凑出完整的逻辑全景图。

## 2. 统一日志协议 (Unified Log Protocol)
所有日志输出（包括 FE 和 BE）必须严格遵循以下 JSON 契约：

| 字段 | 说明 | 类型 | 示例 |
| :--- | :--- | :--- | :--- |
| `ts` | UTC 毫秒时间戳 | `number` | `1740312000` |
| `trace_id` | 全链路唯一标识符 | `string` | `req-550e8400-e29b...` |
| `platform` | 来源平台 | `FE` \| `BE` | `FE` |
| `module` | 业务组件/模块名 | `string` | `swarm.orchestrator` |
| `action` | 具体发生的动作 | `string` | `agent_node_transition` |
| `meta` | 结构化元数据（禁止打散在 msg 中） | `Record` | `{ "agent": "rag_agent" }` |

## 3. 错误分层处理标准 (Error Standard)
所有的 Exception 必须封装入 `AppError` 类：
*   **Infrastructure Error**: 网络、磁盘、数据库。`severity: high`。
*   **Business Error**: 权限不足、非法输入、Agent 推理置信度低。`severity: low/medium`。

## 4. 前端 SourceMap 隐私保护
*   **开发环境**: 开启 inline-sourcemap 方便调试。
*   **生产环境**: `sourcemap: true` 但执行 `post-build.js` 归档。
*   **禁令**: 严禁将 `.map` 文件上传至公网 Nginx 静态服务。

## 5. 调试工具集
*   **Trace ID 提取**: 从 HTTP 响应头 `X-Trace-Id` 获取。
*   **链路复盘**: 使用 `python scripts/trace_analyzer.py --id <trace_id>`。
