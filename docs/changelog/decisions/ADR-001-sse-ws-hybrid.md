# ADR-001: SSE + WebSocket 混合通信方案

| 字段 | 值 |
|------|------|
| **编号** | ADR-001 |
| **日期** | 2026-02-15 |
| **状态** | ✅ 已采纳 |
| **决策者** | 用户 + AI |

## 背景

RAG 系统需要两种通信模式：
1. 用户问答的流式输出（AI 回答逐 token 返回）
2. 系统主动推送（Agent 状态、通知、建议等）

## 决策

采用 **SSE + WebSocket 混合方案**：
- **SSE** 用于对话流式输出（单向、轻量、HTTP 语义清晰）
- **WebSocket** 用于所有主动推送场景（双向、实时、多事件类型）

## 理由

1. SSE 比 WebSocket 更轻量，对话场景不需要双向通信
2. WebSocket 是主动推送的唯一选择
3. 两者各司其职，架构清晰
4. 主流产品（Cursor, ChatGPT）采用类似方案

## 替代方案

1. **纯 WebSocket** — 可行但对话场景过重
2. **纯 SSE** — 无法实现主动推送
3. **轮询** — 实时性差，资源浪费

## 影响

- 前端需要分别管理 SSE 和 WebSocket 连接
- 后端需要 ConnectionManager 管理 WebSocket 连接池
