# DES-HMER-001: Local Edge Engine & Persistent Cache

## 1. 背景与瓶颈 (Evidence from Phase 0)
根据 Phase 0 的 1300 条基线数据审计：
- **P95 延迟失控**: 网络抖动直接导致用户端白屏 > 2s。
- **重复数据拉取**: 每次刷新页面都会重新请求 RAG 列表，造成了不必要的后端负担。

## 2. 核心架构假说 (Hypothesis)
引入 **Local Edge Engine**。通过 `IndexedDB` 存储会话快照与轻量级嵌入向量（如果可能）。
- **同步策略**: 实现 `Incremental Sync (增量同步)` 而非 `Full Replace`。
- **预取逻辑**: 当用户悬停在侧边栏某个会话上时，提前预加载其前 10 条语义片段到内存。

## 3. 技术栈选择
- **Storage**: `idb` (IndexedDB Wrapper).
- **Sync Protocol**: ETag + Last-Modified.
- **State Management**: Zustand Persistence Middleware.
